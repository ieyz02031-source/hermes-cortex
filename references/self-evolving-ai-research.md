# 自进化 AI 系统深度研究

## 研究时间
2026-06-14

## 研究了 6 个核心项目

### 1️⃣ MemOS — 自进化记忆操作系统
- **星数**: 9,853⭐
- **核心价值**: L1/L2/L3 三层记忆，自进化技能
- **性能**: +43.70% 准确率 vs OpenAI Memory，-35.24% token
- **状态**: ❌ 已拒绝（56MB，20MB 依赖，需要 Node.js 20+，固定端口 18800）

### 2️⃣ evolver — GEP 驱动的自进化引擎
- **星数**: 8,667⭐
- **核心价值**: GEP 协议（基因、胶囊、事件），可审计进化
- **性能**: 4,590 次控制试验，45 个科学代码场景
- **状态**: 🔍 待评估

### 3️⃣ Memori — Agent 原生记忆基础设施
- **星数**: 15,283⭐
- **核心价值**: LLM 无关，结构化记忆，81.95% 准确率
- **性能**: 4.97% token 使用（相比全上下文）
- **状态**: ⚠️ 需要云服务

### 4️⃣ engram — 持久化记忆系统
- **星数**: 4,333⭐
- **核心价值**: 单二进制文件，SQLite + FTS5，20 个 MCP 工具
- **性能**: 零依赖，7MB，Git 同步
- **状态**: ✅ 已集成

### 5️⃣ ACE — Agentic Context Engineering
- **星数**: 1,151⭐
- **核心价值**: 三角色架构（生成器、反射器、策展人），增量更新
- **性能**: +10.6% 性能提升，-82.3% 延迟
- **状态**: 🔍 待评估

### 6️⃣ pro-workflow — 自校正记忆工作流
- **星数**: 2,300⭐
- **核心价值**: 自校正记忆，知识平面，质量门禁
- **性能**: -80% 纠正率，34 个技能
- **状态**: ⚠️ 需要 Claude Code

## 优先级建议

| 优先级 | 项目 | 原因 | 预期收益 |
|--------|------|------|----------|
| 🔥 **高** | **engram** | 轻量级，零依赖 | +60% 持久化，10x 检索速度 |
| 🔥 **高** | **evolver** | 自进化能力增强 | +30% 自进化 |
| ⭐ **中** | **ACE** | 上下文优化 | +10% 性能，-82% 延迟 |
| ⭐ **中** | **pro-workflow** | 自校正记忆 | -80% 纠正率 |
| 📚 **低** | **Memori** | 需要云服务 | +40% 准确率 |
| ❌ **拒绝** | **MemOS** | 太重 | +43% 准确率，-35% token |

## 关键发现

1. **轻量级优先**: Go/Rust 单二进制 > Node.js/Python
2. **零依赖优先**: 无依赖 > 有依赖
3. **MCP stdio 优先**: 避免固定端口
4. **engram 已足够**: 不需要 MemOS
5. **自进化是趋势**: evolver、ACE、pro-workflow 都在探索这个方向
