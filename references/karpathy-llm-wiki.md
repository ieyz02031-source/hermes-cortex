# Karpathy LLM Wiki 模式

> 来源: https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f

## 核心思想

别每次都去翻原始资料，而是让 LLM 一点点维护一个长期的 wiki。知识像复利一样累积。

## 架构

```
原始资料
    ↓
┌─────────────────┐
│  LLM 处理       │ ← 抽取实体、概念、关系
└────────┬────────┘
         ↓
┌─────────────────┐
│  结构化存储     │ ← Markdown 文件
└────────┬────────┘
         ↓
┌─────────────────┐
│  知识图谱       │ ← 实体 + 关联
└────────┬────────┘
         ↓
┌─────────────────┐
│  检索与问答     │ ← 基于知识库回答
└─────────────────┘
```

## 关键原则

1. **增量更新** — 不是一次性构建，而是持续更新
2. **结构化存储** — 使用 Markdown 文件，易于阅读和编辑
3. **关联网络** — 通过 wikilinks 建立关联关系
4. **检索增强** — 基于知识库进行精准回答

## 实现方式

### 1. 原始资料存储

```
.raw/
├── source1.pdf
├── source2.md
├── source3.txt
└── ...
```

### 2. 知识库存储

```
wiki/
├── index.md          # 主索引
├── hot.md            # 热缓存
├── log.md            # 操作日志
├── entities/         # 实体笔记
│   ├── person1.md
│   ├── tool1.md
│   └── project1.md
├── concepts/         # 概念笔记
│   ├── concept1.md
│   └── concept2.md
└── sources/          # 来源笔记
    ├── source1.md
    └── source2.md
```

### 3. 知识摄入流程

```
原始资料
    ↓
┌─────────────────┐
│  解析           │ ← 提取文本
└────────┬────────┘
         ↓
┌─────────────────┐
│  抽取           │ ← 实体、概念、关系
└────────┬────────┘
         ↓
┌─────────────────┐
│  创建笔记       │ ← 创建实体/概念笔记
└────────┬────────┘
         ↓
┌─────────────────┐
│  建立关联       │ ← 添加 wikilinks
└────────┬────────┘
         ↓
┌─────────────────┐
│  更新索引       │ ← 更新 index.md
└─────────────────┘
```

### 4. 知识检索流程

```
用户查询
    ↓
┌─────────────────┐
│  理解查询       │ ← 解析意图
└────────┬────────┘
         ↓
┌─────────────────┐
│  检索知识库     │ ← BM25 + 语义搜索
└────────┬────────┘
         ↓
┌─────────────────┐
│  遍历图谱       │ ← 关联推荐
└────────┬────────┘
         ↓
┌─────────────────┐
│  生成回答       │ ← 基于知识库回答
└─────────────────┘
```

## 优势

1. **知识累积** — 像复利一样，知识越积越多
2. **精准回答** — 基于知识库回答，不是训练数据
3. **可追溯** — 每个回答都有来源
4. **可编辑** — 人类可以直接编辑知识库

## 劣势

1. **需要维护** — 需要定期更新和清理
2. **存储成本** — 需要存储空间
3. **构建成本** — 初始构建需要时间

## 应用场景

1. **个人知识管理** — 构建个人知识库
2. **团队知识共享** — 团队共享知识库
3. **研究助手** — 辅助研究工作
4. **学习工具** — 辅助学习和记忆

## 总结

Karpathy LLM Wiki 模式是一种基于 LLM 的知识管理方法，它通过：

1. **增量更新** — 持续更新知识库
2. **结构化存储** — 使用 Markdown 文件
3. **关联网络** — 通过 wikilinks 建立关联
4. **检索增强** — 基于知识库回答

实现了知识的持续累积和智能检索，是构建 AI 第二大脑的有效方法。
